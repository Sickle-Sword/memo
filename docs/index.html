クロス表作成用関数

関数概要

janitorを利用する
表の内容は比教社コースの調査実習のスタイルガイドを参照している（一部未実装）
カイ二乗検定はjanitor，cramerのVはvcdを利用している。 今のところExcelで整形が必要だが，最終的にはRmarkdownから動くようにしたい

my_cross <- function(x){
  crosstab <- 
    x %>% 
    janitor::adorn_totals(where = c('row', 'col')) %>%
    janitor::adorn_percentages(denominator = 'row') %>% 
    dplyr::as_tibble() %>% 
    dplyr::mutate_at(.vars = vars(-1), .funs = ~{round(.*100, 1)})
  
  N <- 
    x %>% 
    janitor::adorn_totals(where = c('row', 'col')) %>%
    dplyr::as_tibble() %>% 
    dplyr::select(N = Total)
  
  cramer <-
    x %>% 
    janitor::untabyl() %>%
    dplyr::select(-1) %>%
    as.matrix() %>%
    vcd::assocstats() %>% 
    .$cramer
  
  p.value <-
    x %>% 
    janitor::chisq.test() %>% 
    .$p.value
  
  dplyr::bind_cols(crosstab, N) %>% 
    dplyr::mutate(cramer = c(cramer, rep(NA_real_, nrow(.) - 1)),
                  p.value = c(p.value, rep(NA_real_, nrow(.) - 1)))
}

カテゴリカル変数の記述統計

関数概要

連続変数の記述統計はpsych::describeなどを使えばいいが，
カテゴリカル変数は良いのがあまりなかったので…
シンプルに度数と割合を算出する
ちなみに関数名のdはdiscrete

describe_d <-
    function(data){
      purrr::map2_dfr(.x = data, .y = names(data),
               .f =  ~{
                 tab <- janitor::tabyl(.x)
                 variable_name <- .y
                 N <- tab[,2] %>% sum()
                 dplyr::tibble(
                   variables = c(variable_name,
                                 stringr::str_c('  ', tab[,1])),
                   N = c(N, tab[,2]),
                   percent = c(1, tab[,3]))
                 })
      }

reticulate

  • R上でpythonを動かせるパッケージ
  • VScodeとかpython用にエディタとかを入れてみたものの，結局プロジェクト管理とかが面倒になってしまったので，全部Rstudio上でできるようにしたかった
library(reticulate)

使い方

1. python環境の指定

  • 狙ったpythonのバージョンを指定する
  • ここで結構ハマった。なんでも一度何らかの関数でpythonを動かしてしまうと， 一度Restart sessionしない限りpythonの環境を変更できないらしい。
  • むしろlibrary(reticulate)よりも前にこれを実行しておいた方がいいかもしれない

正規表現の仕様変更（R 4.0.0～）

調べても意外と情報が得られなかったので一応メモ

従来の仕様

例えば

a <- 'aabb ;) 334'

のような表現にマッチさせるには，

stringr::str_extract(a, 'aabb ;) 334')

これではだめで，

stringr::str_extract(a, 'aabb ;\\) 334')
[1] "aabb ;) 334"

のように正規表現を書かなければならない

'()'や'.'といった正規表現中で特別な意味を持つ文字をもともとの形でマッチさせるには，文字列の前にエスケープ用のバックスラッシュ'\'をつける必要があるが，'\'自体も特殊な文字扱いなので，最終的に'\\.'のような形になる。

バックスラッシュ2つ'\\'にマッチさせるなら，正規表現は'\\\\'になる。これは面倒。

新しい仕様

pythonなどで実装されているraw stringという正規表現の書き方が使えるようになった。r'...'のように書くと，'...'の中でバックスラッシュが特別扱いされなくなる。
すなわち，以下のコードでよくなる。

stringr::str_extract(string = a, pattern = r'(aabb ;\) 334)')
[1] "aabb ;) 334"

ここで注意すべきは，特別扱いがなくなるのはバックスラッシュのみなので，')'にマッチさせるにはその前にバックスラッシュを置く必要がある点。

ちなみに，pythonだとr'...'でよいが，Rだとr'(...)'というように正規表現全体を括弧で括る必要がある。なんか新しい問題が起こりそうで恐い。

# python
import re

m = re.match(pattern = r'aabb ;\) 334', string = r.a)
m.group()
'aabb ;) 334'
