クロス表作成用関数

関数概要

janitorとgtを利用する
表の形式は調査実習のスタイルガイドを参照
カイ二乗検定はjanitor::chisq.test，cramerのVはvcd::assocstatsを利用している。

my_cross <- function(.data, .x, .y){
  .x <- rlang::enquo(.x)
  .y <- rlang::enquo(.y)
  
  .contents_.y <- dplyr::pull(.data, !!.y) 
  if(any(class(.contents_.y) == 'factor')){
    .contents_.y <- 
      forcats::fct_unique(.contents_.y) %>% 
      as.character()
  } else {
    .contents_.y <- 
      factor(.contents_.y) %>% 
      forcats::fct_unique() %>% 
      as.character()
  }
  
  .tabyl <- janitor::tabyl(.data, !!.x, !!.y) 
  
  N <- 
    .tabyl %>% 
    janitor::adorn_totals(where = c('row', 'col')) %>%
    dplyr::as_tibble() %>% 
    dplyr::pull(Total) %>% 
    # 合計のところの括弧
    stringr::str_c('（', ., '）')
  
  .p.value <-
    janitor::chisq.test(.tabyl) %>% 
    .$p.value
  
  .cramer <-
    .tabyl %>% 
    janitor::untabyl() %>%
    dplyr::select(-1) %>%
    as.matrix() %>%
    vcd::assocstats() %>% 
    .$cramer 
  
  .crosstab_raw <- 
    .tabyl %>% 
    janitor::adorn_totals(where = c('row', 'col')) %>%
    janitor::adorn_percentages(denominator = 'row') %>% 
    dplyr::as_tibble() %>% 
    dplyr::mutate(dplyr::across(.cols = where(is.numeric),
                                .fns = ~{round(.*100, digits = 1)}
                                )) %>% 
    # パーセント記号
    dplyr::mutate(across(.cols = 1,
                         .fns = ~{stringr::str_c(., '（％）')}
                         )) %>% 
    dplyr::mutate(N = N)
  
  # gtによる整形
  .crosstab_gt <-
    gt::gt(.crosstab_raw) %>% 
    gt::tab_spanner(label = rlang::as_label(.y),
                    columns = .contents_.y) %>% 
    gt::tab_source_note(stringr::str_interp("Cramer's V = $[.3f]{.cramer}   p = $[.4f]{.p.value}"))
    
  return(.crosstab_gt)
}

カテゴリカル変数の記述統計

関数概要

連続変数の記述統計はpsych::describeなどを使えばいいが，
カテゴリカル変数は良いのがあまりなかったので…
シンプルに度数と割合を算出する
ちなみに関数名のdはdiscrete

describe_d <- function(.data, ...){
  .data <- select(.data, ...)
  purrr::map2_dfr(.x = .data, .y = names(.data),
                  .f =  ~{
                    tab <- janitor::tabyl(.x)
                    variable_name <- .y
                    N <- tab[,2] %>% sum()
                    dplyr::tibble(
                      variables = c(variable_name, stringr::str_c('  ', tab[,1])),
                      N = c(N, tab[,2]),
                      percent = c(1, tab[,3]))
                  })
  }

正規表現の仕様変更（R 4.0.0～）

調べても意外と情報が得られなかったので一応メモ

従来の仕様

例えば

a <- 'aabb ;) 334'

のような表現にマッチさせるには，

stringr::str_extract(a, 'aabb ;) 334')

これではだめで，

stringr::str_extract(a, 'aabb ;\\) 334')
[1] "aabb ;) 334"

のように正規表現を書かなければならない

'()'や'.'といった正規表現中で特別な意味を持つ文字をもともとの形でマッチさせるには，文字列の前にエスケープ用のバックスラッシュ'\'をつける必要があるが，'\'自体も特殊な文字扱いなので，最終的に'\\.'のような形になる。

バックスラッシュ2つ'\\'にマッチさせるなら，正規表現は'\\\\'になる。これは面倒。

新しい仕様

pythonなどで実装されているraw stringという正規表現の書き方が使えるようになった。r'...'のように書くと，'...'の中でバックスラッシュが特別扱いされなくなる。
すなわち，以下のコードでよくなる。

stringr::str_extract(string = a, pattern = r'(aabb ;\) 334)')
[1] "aabb ;) 334"

ここで注意すべきは，特別扱いがなくなるのはバックスラッシュのみなので，')'にマッチさせるにはその前にバックスラッシュを置く必要がある点。

ちなみに，pythonだとr'...'でよいが，Rだとr'(...)'というように正規表現全体を括弧で括る必要がある。なんか新しい問題が起こりそうで恐い。

# python
import re

m = re.match(pattern = r'aabb ;\) 334', string = r.a)
m.group()
'aabb ;) 334'
